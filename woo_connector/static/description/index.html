Sync direction clarity

Clear WooCommerce to Odoo and Odoo to WooCommerce flow

One thing many competitor pages miss is direct explanation of sync direction. They say “bi-directional” but do not clearly show what moves where. This layout keeps both directions separate, which helps reduce confusion and improves buyer trust.

W→O

WooCommerce to Odoo flow

Use Odoo as the operational review layer for storefront data.

  • Import products from WooCommerce into Odoo sync models
  • Pull customer records and order data from WooCommerce
  • Sync categories and coupons from WooCommerce
  • Refresh inventory information from WooCommerce product stock
  • Receive webhook-based updates for product, order, customer, category, and coupon changes
  • Write sync outcomes into reports and status logs
O→W

Odoo to WooCommerce flow

Push selected operational changes back to the store when needed.

  • Update selected products from Odoo to WooCommerce
  • Create or update customers in WooCommerce
  • Create or update categories in WooCommerce
  • Create or update coupons in WooCommerce
  • Push selected order updates such as status and customer note
  • Use mapping controls to shape how data is prepared before sync

FAQ section

Questions buyers often ask before installing

Does the connector support multiple WooCommerce stores?

Yes. The module uses an instance-based structure, so multiple WooCommerce stores can be managed separately inside Odoo.

Can I import data from WooCommerce into Odoo?

Yes. The current module structure supports import workflows for products, orders, customers, categories, coupons, and inventory data.

Can I push data from Odoo back to WooCommerce?

Yes. Selected product, customer, category, coupon, and order updates can be pushed from Odoo to WooCommerce.

Does the connector support automation?

Yes. The app includes scheduled sync mechanisms and webhook handling for faster update processing.

Can I customize field behavior?

Yes. The field mapping framework allows mapping by entity, sample payload inspection, and test-based validation.

How can users track sync issues?

Sync reports, webhook logs, dashboard metrics, and sync status records are part of the current module workflow.
